SYSTEM OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Three-Phase Workflow:

PHASE 1: FETCH (bird CLI + Link Expansion)
  Input:  Twitter cookies (auth_token, ct0)
  Action: Fetch bookmarks, expand t.co links, extract content
  Output: .state/pending-bookmarks.json (enriched metadata)
  Time:   ~10 seconds for 20 bookmarks

PHASE 2: PROCESS (Claude Code Analysis)
  Input:  pending-bookmarks.json
  Action: Claude analyzes, generates titles, categorizes, files
  Output: bookmarks.md + knowledge/ files
  Time:   ~30 seconds for 20 bookmarks (parallel Haiku for speed)

PHASE 3: TRIAGE (Interactive Categorization)
  Input:  bookmarks.md archive
  Action: Interactive questions → categorization → action planning
  Output: triage.json with actionable items
  Time:   ~1 minute per 20 bookmarks (interactive)

Triage Categories:
  - dev-tool: Tools/frameworks to integrate
  - library: Code patterns to vendor
  - idea: Concepts to implement
  - integration: APIs to connect
  - ignore: Not relevant
